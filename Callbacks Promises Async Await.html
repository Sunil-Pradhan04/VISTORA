📚 NOTES FOR YOUR STUDY

1. 🔁 Callbacks

✅ Definition:

A function passed into another function as an argument and executed later.

✅ Syntax:

js
function fetchData(callback) {
  setTimeout(() => {
    callback("Data received");
  }, 1000);
}

fetchData((data) => {
  console.log(data);
});

✅ Problems with Callbacks:

  • Callback Hell (nested callbacks)

  • Hard to read/maintain

  • Difficult to handle errors

2. 📦 Promises

✅ Definition:

An object representing the eventual completion or failure of an async operation.

✅ States:

  • Pending

  • Fulfilled (resolved)

  • Rejected

✅ Syntax:

js
const fetchData = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve("Data fetched");
    }, 1000);
  });
};

fetchData().then(data => console.log(data)).catch(err => console.log(err));

✅ Chaining:

js
fetchData()
  .then(data => {
    console.log(data);
    return "More data";
  })
  .then(more => console.log(more));

3. ⚡ Async/Await

✅ Definition:

Syntactic sugar over Promises for writing async code in a synchronous style.

✅ Syntax:

js
const fetchData = () => {
  return new Promise(resolve => {
    setTimeout(() => resolve("Data received"), 1000);
  });
};

const getData = async () => {
  const data = await fetchData();
  console.log(data);
};

getData();

✅ Error Handling:

js
const getData = async () => {
  try {
    const data = await fetchData();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
};

⚡ Async/Await

✅ Simplified Definition:

async/await lets you write asynchronous code that looks and behaves like synchronous code, making it easier to read and manage.

  • async makes a function always return a Promise.

  • await pauses the function until the Promise resolves or rejects.

🧠 Think of it like this:

Instead of chaining .then() with Promises, you can use await to "pause" and get the result as if you're writing normal step-by-step code.
